# VaultPi — سجل التتبع الرئيسي (يُقرأ ويُحدَّث في بداية كل جلسة)

> نفس أسلوب عملنا على دابيا: عميل Supabase مباشر + RLS، مفاتيح عامة في الكود،
> بادئة `vaultpi` للملفات، وترقيم للملفات (`vaultpi_01`, `vaultpi_02`…).

## نظرة عامة على التطبيق
**PiVault** — خزنة آمنة على شبكة Pi لحفظ: الملفات بكل أنواعها، الصور،
الفيديو، الروابط، و**المذكرات**. شبيه بـ"صور Google" لكن أشمل، مع **اختصار
عائم على حافة الشاشة** قابل للسحب والتخصيص.

- الواجهة: Next.js 15 + React 19 + Tailwind v4 + shadcn/ui (من Pi App Studio).
- الخلفية: **Supabase** (Postgres + Storage + Edge Function) — مشروع `VaultPi`
  (`shsuznbuaxkkykvgbklb`).
- الدخول: **Pi Network** → دالة جسر تُنشئ جلسة Supabase حقيقية → بعدها كل
  الوصول عميل مباشر محميّ بـ RLS.

## ⚠️ بنود متبقية (تحتاج قراراً أو خطوة خارجية)
1. **التشفير الحقيقي من طرف العميل (E2E / Zero-Knowledge):** الواجهة تعرض
   شارة "AES-256/Encrypted"، والحالة الحقيقية الآن: الملفات مخزّنة في bucket
   **خاص** ومشفّرة **at-rest** من Supabase، ومعزولة لكل مستخدم عبر RLS —
   حماية حقيقية لكنها **ليست zero-knowledge**. لتحقيق تشفير حقيقي قبل الرفع
   يلزم تشفير البايتات في المتصفح (WebCrypto) بمفتاح مشتق من المستخدم. **بند
   مستقبلي**، ويُذكر للمستخدم بصراحة.
2. **تسجيل النطاق في Pi Developer Portal:** لكي تختفي علامة ⚠️ ويعمل الدفع،
   يجب نشر التطبيق على نطاق ثابت ووضع `validation-key.txt` عليه وتسجيله في
   بوابة مطوّري Pi. (خطوة خارجية بعد النشر).
3. **المدفوعات:** بنية Pi payment موجودة (`lib/pi-payment.ts` + دوال Netlify)
   لكنها ما زالت تشير لخلفية App Studio؛ الخزنة لا تحتاج دفعاً حالياً. عند
   الحاجة (مثلاً ترقية المساحة) تُنقل للدوال الخاصة.
4. **حصة التخزين الفعلية:** الحد المبدئي 5GB لكل مستخدم (`storage_quota`)،
   لكن باقة Supabase المجانية تعطي 1GB إجمالي للمشروع — للتوسّع تلزم ترقية
   الباقة.
5. **الوسوم (Tags):** الطبقة الخلفية جاهزة (`tags`, `item_tags`, `setItemTags`)
   وتظهر في AI؛ ينقص واجهة "إضافة وسم لعنصر" من قائمة العنصر.

## معلومات المشروع
- Supabase project: `VaultPi` = `shsuznbuaxkkykvgbklb` (eu-central-1)
- Edge function: `vaultpi-auth` (verify_jwt=false)
- الفرع: `claude/vaultpi-app`
- المفاتيح العامة مكتوبة في `lib/vaultpi/config.ts` (آمنة للنشر).

## قواعد ثابتة (مثل دابيا)
1. أي عطل RLS: افحص هل الجدول `public.X` أم `storage.objects` — نوعان مختلفان.
2. لا تُغلق بنداً كـ ✅ إلا بعد فحص الكود الفعلي والبناء.
3. أي دالة SQL جديدة: `set search_path = ''` (توصية أمان).
4. كل ميزة جديدة = واجهة + اتصال Supabase حقيقي + RLS + فحص بناء.
